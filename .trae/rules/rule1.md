一、项目定位
本项目为上海迪士尼舆情全自动分析系统，实现：关键词输入 → 多平台爬取 → 数据清洗 → AI 情感分析 → 可视化展示 → 智能报告一键生成。
二、流程规范
全流程自动化
入口统一：run_full_analysis.py
顺序：爬取 → 合并 → 清洗 → 情感分析 → 可视化 → 报告 → 前端启动
全程无需人工干预，异常可断点续跑。
数据采集
平台：微博、知乎、贴吧、虎扑
微博必须支持：官方账号进入、帖子正文提取、评论区展开、评论完整爬取、穿透 iframe
技术：仅使用 Selenium + Edge 驱动
输出：data/raw/ 下按平台保存规范 CSV
数据处理
统一字段、统一编码、空值填NULL
清洗：去重、去表情、过滤短文本（≥10 字）
合并输出：data/merged_all_platform.csv
情感分析：优先 DeepSeek API，失败自动降级 SnowNLP
分析与可视化
输出：data/analyzed_comments.csv
图表：情感分布、热度、高频词、维度分析
前端：Streamlit (app.py)，界面为舆情驾驶舱
图表保存到 data/viz/
智能问答与报告
基于 RAG 从已分析数据中检索回答
自动生成学术报告：data/academic_report_*.md
三、架构与规范
目录结构严格按文档执行
代码模块化、复用基类与工具函数
控制台彩色日志，清晰显示进度 / 错误
所有异常必须捕获，不中断主流程
四、交付要求
代码可直接运行、可替换、不破坏原有结构
输出完整可运行代码，而非片段
问题定位精准：说明原因→给出修复→验证步骤
一切修改服务于：稳定爬取、完整评论、全自动、可展示
