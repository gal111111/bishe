# 公共设施满意度分析系统 - 技术架构文档

## 📋 项目概述

**项目主题**：基于多源数据的公共设施满意度分析与预警系统

**核心目标**：
1. 从知乎、微博等平台爬取公共设施相关的用户评论
2. 对评论进行情感分析和满意度评分
3. 建立公共设施满意度预警机制
4. 提供多维度的分析结果和可视化展示

## 📊 详细模块说明

### 1. 数据采集层

#### 1.1 知乎爬虫 (zhihu_selenium_spider.py)
- **技术栈**：Selenium + Edge WebDriver
- **核心功能**：
  - 自动化登录知乎（使用Cookie）
  - 关键词搜索相关话题
  - 爬取回答和评论
- **反爬策略**：
  - Cookie持久化
  - 随机延迟
  - User-Agent伪装

#### 1.2 微博爬虫 (weibo_api_spider.py)
- **技术栈**：Requests + 微博API
- **核心功能**：
  - Cookie验证和管理
  - 微博内容获取
  - 评论爬取
- **数据保存**：CSV格式

#### 1.3 数据格式化 (data_formatter.py)
- **功能**：统一多源数据格式
- **输出字段**：
  - `content`: 评论内容
  - `facility_type`: 设施类型
  - `region`: 地区
  - `source`: 数据来源（知乎/微博）

#### 1.4 AI搜索优化 (ai_search_optimizer.py)
- **技术栈**：DeepSeek-V3大模型
- **核心功能**：
  - 自动优化搜索关键词
  - 生成更相关的搜索词
  - 提高爬取效率和数据质量
- **优化策略**：
  - 使用DeepSeek-V3生成语义相关的搜索词
  - 限制优化词数量以确保爬取速度
  - 按相关性排序搜索词

#### 1.5 并行处理模块
- **技术栈**：Python ThreadPoolExecutor
- **核心功能**：
  - 并行爬取不同平台的数据
  - 提高爬取速度
  - 优化资源利用
- **实现方式**：
  - 使用ThreadPoolExecutor同时爬取知乎和微博
  - 每个平台使用独立线程
  - 线程安全的数据收集

### 2. 数据预处理层

#### 2.1 数据清洗模块 (data_cleaning.py)

**问卷数据清洗算法**：
```
算法：缺失值处理 + 异常值检测 + 数据标准化

1. 缺失值处理
   - 数值型（满意度得分）：中位数填充（对异常值鲁棒）
   - 数值型（年龄、使用频率）：均值填充
   - 类别型（设施类型、地区）：众数填充

2. 异常值检测 - IQR四分位数法
   - Q1 = 25%分位数，Q3 = 75%分位数
   - IQR = Q3 - Q1
   - 异常值边界：[Q1 - 1.5×IQR, Q3 + 1.5×IQR]
   - 处理方式：截断至边界值（保留样本）

3. 数据标准化
   - Min-Max归一化：将数据缩放到[0,1]区间
   - Z-score标准化：均值为0，标准差为1
```

**文本反馈清洗算法**：
```
算法：特殊符号过滤 + 停用词删除 + 中文分词

1. 文本预处理
   - 移除HTML标签、URL、邮箱
   - 移除多余空白字符
   - 只保留中英文数字

2. 中文分词 - jieba
   - 精确模式分词
   - 过滤停用词（自定义停用词表）
   - 过滤单字符词

3. 文本长度过滤
   - 去除长度<5的文本（无意义内容）
```

**时序数据清洗算法**：
```
算法：时间格式标准化 + 异常值处理（Z-score法）

1. 时间格式标准化
   - 自动识别时间列
   - pd.to_datetime转换
   - 按时间排序

2. 异常值检测 - Z-score法
   - Z = (X - μ) / σ
   - 阈值：|Z| > 3视为异常
   - 处理：截断至边界值

3. 时间连续性处理
   - 重复时间戳：添加1ms偏移
```

#### 2.2 非评论元素过滤模块

**非评论元素过滤算法**：
```
算法：规则过滤 + AI过滤

1. 规则过滤
   - 检查是否包含非评论元素的特征
   - 非评论元素模式：
     - "发布于 ", "赞同", "添加评论", "分享", "收藏", "喜欢", "收起"
     - "关注", "粉丝", "评论", "点赞", "转发", "回复", "编辑"
     - "编辑于 ", "赞同 ", "评论内容", "评论作者", "评论时间", "评论点赞"

2. AI过滤
   - 使用DeepSeek-V3大模型进行更精确的过滤
   - 判断文本是否为真实的用户评论
   - 过滤系统生成的操作按钮、时间戳等

3. 过滤策略
   - 规则过滤作为第一道防线
   - AI过滤作为第二道防线
   - 严格过滤，确保数据质量
```

### 3. 分析挖掘层

#### 3.1 情感分析模块 (sentiment_analysis.py)

**DeepSeek大模型情感分析**：
- **模型**：deepseek-ai/DeepSeek-V3（硅基流动API）
- **API端点**：https://api.siliconflow.cn/v1/chat/completions
- **分析维度**：
  - 情感极性：积极/中性/消极
  - 情感得分：-1.0 ~ 1.0
  - 情绪类别：喜/怒/哀/惧/惊/恶
  - 方面级情感分析（ABSA）

**CSI（City Satisfaction Index）满意度模型**：
```
CSI = 情感得分 × 紧急度 × 情感修饰因子

- 情感得分：-1（极不满意）~ 1（非常满意）
- 紧急度：根据问题严重程度动态调整
- 情感修饰因子：根据情绪类别调整
- 最终归一化到：0 ~ 100分
```

**多Agent架构**：
- **PreprocessAgent**: 文本预处理
- **ExtractorAgent**: 关键信息抽取
- **RuleCheckerAgent**: 规则基线检查
- **ReflectionAgent**: 结果反思优化
- **AdvisorAgent**: 建议生成

**规则基线模型**（API fallback）：
- 基于情感词典的关键词匹配
- 正向词库、负向词库
- 简单统计方法

#### 3.2 时序分析模块 (timeseries_analysis.py)

**ARIMA短期趋势预测**：
```
算法：AutoRegressive Integrated Moving Average

1. 平稳性检验 - ADF检验
   - H0：非平稳
   - p-value < 0.05：拒绝H0，认为平稳

2. 差分处理
   - 若不平稳，进行d阶差分

3. 参数选择
   - p：自回归阶数（ACF图）
   - d：差分阶数
   - q：移动平均阶数（PACF图）

4. 模型训练与预测
```

**Prophet含节假日效应预测**：
```
算法：Facebook Prophet

特点：
- 自动处理缺失值
- 支持节假日效应
- 自动检测变点
- 可解释性强

组件：
- 趋势项：分段线性/逻辑增长
- 季节性：年、周、日
- 节假日：自定义假期
- 误差项：高斯分布
```

**LSTM长时序预测**：
```
算法：Long Short-Term Memory神经网络

架构：
- 输入层：(batch_size, lookback, 1)
- LSTM层1：50个神经元，return_sequences=True
- LSTM层2：50个神经元
- Dense层1：25个神经元
- Dense层2：1个神经元（输出）

训练：
- 优化器：Adam
- 损失函数：MSE
- 数据标准化：MinMaxScaler [0,1]
```

**变点检测 - PELT算法**：
```
算法：Pruned Exact Linear Time

目标：检测满意度时间序列中的突变点

特点：
- 计算效率高（线性时间）
- 可调整惩罚参数
- 支持多种成本函数

简化版（无ruptures库）：
- 滑动窗口方差检测
- 窗口大小：7天
- 阈值：0.15
```

#### 3.3 主题发现模块 (topic_discovery.py)

**LDA主题模型**：
```
算法：Latent Dirichlet Allocation

原理：
- 文档-主题分布：Dirichlet(α)
- 主题-词分布：Dirichlet(β)
- 每个文档由多个主题混合生成
- 每个主题由多个词混合生成

参数：
- K：主题数量
- α：文档-主题稀疏性
- β：主题-词稀疏性
```

**BERTopic**：
- 基于BERT的主题建模
- 语义级主题发现
- 更先进的主题聚类

#### 3.4 RAG增强模块 (rag_support.py)

**Sentence-BERT语义检索**：
```
算法：Sentence-BERT + 余弦相似度

流程：
1. 文档向量化：Sentence-BERT编码
2. 索引构建：FAISS/HNSW
3. 查询编码：用户问题向量化
4. 相似度计算：余弦相似度
5. 结果检索：Top-K最相关文档

优化：
- TF-IDF特征融合
- 缓存机制
- 批量处理
```

### 4. 预警决策层

#### 4.1 预警系统模块 (alert_system.py)

**静态预警 - 阈值判定法**：
```
算法：固定阈值判定

阈值设置：
- 黄色预警：满意度 < 60分
- 红色预警：满意度 < 40分

触发条件：
if current_avg < red_threshold:
    红色预警（等级3）
elif current_avg < yellow_threshold:
    黄色预警（等级2）
else:
    无预警
```

**动态预警 - Isolation Forest异常检测**：
```
算法：Isolation Forest（孤立森林）

原理：
- 异常点更容易被孤立
- 通过随机切分构建孤立树
- 异常点路径长度更短

参数：
- contamination：异常比例（0.1）
- random_state：随机种子（42）

判定：
- 预测值 = -1：异常
- 预测值 = 1：正常
```

**组合预警 - 双重触发机制**：
```
算法：情感倾向 + 满意度得分 双重触发

条件：
负面评论比例 ≥ 0.7  AND  满意度得分 ≤ 50

预警等级：
- 红色：满意度 < 30 或 负面比例 > 0.8
- 黄色：满意度 < 50 或 负面比例 > 0.7
- 蓝色：其他情况
```

**自适应阈值 - 基于历史统计**：
```
算法：历史均值 ± k × 历史标准差

统计量：
- 历史均值：μ
- 历史标准差：σ
- 置信度：95% → k = 1.96
          99% → k = 2.58

阈值计算：
- 黄色阈值 = max(μ - k×σ, 30)
- 红色阈值 = max(μ - 1.5×k×σ, 20)

优势：
- 适应不同设施的基准差异
- 考虑历史数据分布
- 避免固定阈值的局限性
```
