# 评论爬取完整性问题分析与修复

## 问题描述
用户反馈：原有四个平台（微博、知乎、贴吧、虎扑）的爬虫存在评论爬取不完整的问题：
- 爬完帖子的第一个评论之后，打开第一个评论的回复
- 爬完之后就去爬第二个评论的内容和他的回复
- 然后就没有了，没有把整个评论区爬下来
- 爬下来的评论数不等于这一条帖子的评论数

## 问题分析

### 1. 微博爬虫 (weibo_spider.py)
**当前问题：**
- 滚动加载次数：30次（第419行）
- "查看更多"按钮只点击一次（第435-457行）
- 点击后只滚动20次（第450-452行）
- 没有处理多层回复的展开

**需要修复：**
- 增加滚动加载次数
- 多次尝试点击"查看更多"
- 展开所有回复评论

### 2. 知乎爬虫 (zhihu_selenium_spider.py)
**当前问题：**
- 滚动加载次数：30次（第445行）
- 主要提取回答，而不是所有评论
- 没有处理多层嵌套的评论

**需要修复：**
- 增加滚动加载次数
- 改进评论提取逻辑
- 处理嵌套回复

### 3. 贴吧爬虫 (tieba_selenium_spider.py)
**当前问题：**
- 滚动加载次数：30次（第389行）
- "下一页"只点击一次（第405-426行）
- 点击后只滚动20次（第419-421行）

**需要修复：**
- 循环点击"下一页"直到没有更多
- 增加每一页的滚动次数
- 处理楼中楼回复

### 4. 虎扑爬虫 (hupu_selenium_spider.py)
**当前问题：**
- 滚动加载次数：30次（第552行）
- "展开全部回复"只点击一次（第523-547行）
- "下一页"只点击一次（第568-589行）

**需要修复：**
- 循环点击"下一页"
- 展开所有回复
- 增加滚动加载次数

## 修复策略

### 通用修复策略：
1. **增加滚动加载次数**：从30次增加到50-100次
2. **循环点击"下一页"/"查看更多"**：直到没有更多内容
3. **展开所有回复**：点击所有"展开回复"按钮
4. **更长的等待时间**：确保内容加载完成
5. **多次提取尝试**：滚动后多次尝试提取评论

### 各平台特定策略：

#### 微博
- 多次点击"查看更多评论"
- 展开所有回复
- 更长的滚动加载

#### 知乎
- 改进评论提取选择器
- 处理嵌套回复
- 更长的滚动

#### 贴吧
- 循环点击"下一页"
- 处理楼中楼
- 多次滚动

#### 虎扑
- 循环点击"下一页"
- 展开所有回复
- 更长的滚动

## 修复后的预期效果
- 每个帖子能爬取90%以上的评论
- 包含主评论和所有回复
- 评论数量接近页面显示的评论数
